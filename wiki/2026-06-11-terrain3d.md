# terrain3d — Godot 3.5 Procedural 3D Terrain

## Summary / 摘要

terrain3d is an open-source Godot 3.5 project that demonstrates procedurally generated 3D terrain using GDNative bindings to C++ for performance-critical generation code. It divides the world into terrain tiles at multiple LOD (Level of Detail) levels and stitches adjacent tile meshes together to prevent visible seams. The project is unfinished and no longer maintained, but provides a reference implementation of LOD terrain stitching in Godot. The codebase is 49.9% GDScript, 31.2% C++, 10.6% Python (build tooling), and 8.3% GLSL.

## Content / 内容

### Architecture

The terrain is divided into square tiles arranged in a grid. Each tile generates its own mesh using noise-based height data computed via GDNative C++ code. As the camera approaches a tile, it splits into four child tiles at a higher LOD (finer detail). The inverse occurs as the camera moves away — child tiles merge back into their parent.

### LOD Stitching

A key technical challenge solved by this project is stitching mesh boundaries between neighboring tiles at different LOD levels. When a tile splits but its neighbor does not, the boundary edge has vertices at different resolutions. The project adjusts the higher-resolution edge to align with the coarser neighbor, preventing visible cracks in the terrain surface.

### Implementation Details

- **Engine**: Godot 3.5
- **Native code**: C++ via GDNative (godot-cpp bindings)
- **Shader code**: GLSL for GPU-side experiments (`gpu_experiment/`)
- **Build system**: SCons
- **Player controls**: WASD movement, Escape to toggle mouse capture

### Known Bug

Occasionally a parent terrain tile remains visible even after being split into child tiles, causing overlapping geometry.
