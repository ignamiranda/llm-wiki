# Procedural Sword Generator — Caius Walker BSc Honours Project

## Summary

Caius Walker's BSc Honours project (Computer Games Technology) designed and implemented a procedural sword generator in C#/Unity. It selects random sword parts from CSV data files, generates 3D meshes from vertex position calculations, and computes physics-based damage using kinetic energy, point of balance, and target mass.

## Content

### Architecture

The prototype was implemented as a Unity package for reusability. Sword part data (blade types, guards, grips, pommels) is stored in CSV files, chosen over JSON for smaller file size and easier deserialisation into Unity.

### Mesh Generation

Each sword part's mesh is generated by calculating vertex positions from the part's length, width, and thickness parameters defined in CSV data. A brute-force approach was used initially, with plans to replace it with a more sophisticated algorithm.

### Physics-Based Damage

The system calculates:
- **Volume**: Per-part volume from mesh cross-sections, summed for total
- **Point of balance**: Derived from volume distribution
- **Mass**: From total volume and material density
- **Kinetic energy**: Before and after impact, factoring sword weight, speed, impact point along the blade, and target weight

This enables dynamic damage that responds to design changes — longer blades weigh more, hit with different balance points, and deal damage differently than shorter blades.

### CSV Data Format

CSV files store dimensional parameters for each sword component. Walker noted that CSV was chosen for its compact footprint and straightforward Unity deserialisation.

### Known Issues

- Curve vertex positions incorrectly calculated in some cases
- Difficulty entering correct part data
- Kinetic energy calculation errors before/after impact

### Continued Work

A second prototype is in development addressing:
- JSON replacement for CSV (with JSON-to-CSV converter for memory efficiency)
- Simplified, more flexible structs for part positioning
- Improved mesh generation algorithm
- Corrected kinetic energy calculations
- Standalone visual sword viewer outside Unity Editor

## Key Takeaways

- First documented academic project (BSc Honours) on procedural sword generation
- Novel integration of physics-based damage calculation with procedural mesh generation
- CSV-based part data enabling designer-friendly extensibility
- Demonstrates feasibility of historically/fictionally-informed procedural weapon design
