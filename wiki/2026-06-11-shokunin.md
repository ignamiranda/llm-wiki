# Shokunin — 62 Engineering Skills for AI Coding Agents

## Summary / 摘要

Shokunin (职人, Japanese for "artisan") is an open-source ecosystem by [[elias-oulkadi]] that packages 62 engineering skills, ChromaDB-based persistent memory, and MCP servers into a single-command install for AI coding runtimes. Skills span 10 domains from infrastructure (Docker, Kubernetes, Terraform) to frontend (component design, motion, aesthetics) to system functionality (memory, updates, planning). The ecosystem is runtime-agnostic with native support for [[agent-skills|OpenCode]], Claude Code, Cursor, Windsurf, Cline, and Continue.dev.

## Content / 内容

### Architecture

The Shokunin ecosystem consists of four layers:

1. **Skills** — 62 SKILL.md files in 10 domains, each with trigger-optimized descriptions, procedural workflows, error handling, production checklists, anti-patterns, and cited sources. CI-validated on every push.

2. **Memory** — ChromaDB-backed persistent context using multi-strategy recall: vector similarity + BM25 + temporal decay + reciprocal rank fusion (RRF). Features freshness decay (30-day half-life) and claim verification via `verify_file_path` MCP tool.

3. **MCP Servers** — 9 MCP tools: `store_context`, `search_context`, `get_session_summary`, `multi_search_context`, `consolidate_memories`, `list_sessions`, `continue_session`, `save_message`, `verify_file_path`.

4. **Self-Updates** — Declarative manifest with drift detection. Weekly cleanup and memory backup via Windows Task Scheduler or Linux cron.

### Multi-Runtime Compatibility

Shokunin is runtime-agnostic. The core (skills, memory, scripts) works across six runtimes with different integration methods:

- **OpenCode** — native, built-in config template
- **Claude Code** — reads SKILL.md files, memory via MCP
- **Cline (VS Code)** — reads SKILL.md, memory via MCP
- **Cursor** — reads SKILL.md, memory via rules
- **Continue.dev** — reads SKILL.md, memory via rules
- **Windsurf** — reads SKILL.md, memory via rules

### Memory System

The v4.2.3 memory system implements multi-strategy recall combining vector search (ChromaDB), BM25 keyword matching, temporal decay (exponential recency blending with 30-day half-life), and reciprocal rank fusion. Session management supports explicit session continuation via `chroma-helper.py`. All data is stored locally at `~/.shokunin/memory/` — no cloud, no telemetry.

## Key Takeaways / 关键收获

- Shokunin is the largest known curated collection of AI agent engineering skills (62), distinct from [[2026-06-11-mattpocock-skills|Matt Pocock's skills repository]] in scope (engineering domains vs TypeScript/testing focus) and approach (ChromaDB memory + MCP servers as integral components).
- The CI-validated skill format (frontmatter, workflow, error handling, sources) is a pattern for quality assurance in agent skill development.
- Multi-strategy recall with freshness decay addresses the "stale memory" problem in persistent AI agent context.
