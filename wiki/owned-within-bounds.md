# Owned Within Bounds

## Definition

The "owned within bounds" principle is a pattern used in [[layer-based-procedural-generation]] to resolve which chunk gets to store a piece of data when that data overlaps multiple chunk boundaries. For example, a path or connection that crosses from one chunk into a neighboring chunk could logically belong to either. By applying strict ownership rules (e.g., the chunk whose center is closest, or the chunk that contains the data's origin point), each piece of cross-boundary data is assigned to exactly one chunk, avoiding duplication and conflicts.

## Key Properties

- Resolves ownership of cross-boundary data (paths, connections, regions)
- Ensures each piece of data is stored by exactly one chunk
- Complements the "overlapping bounds" principle (collect all data that overlaps a chunk's bounds)
- Essential for determinism when multiple chunks could claim the same data

## Examples

- In The Cluster example from [[layerprocgen]], connections between locations that span multiple LocationLayer chunks are assigned to a single chunk using this principle
- Regions generated by a RegionLayer use this to avoid overlapping region claims
- A path that crosses three chunk boundaries is stored by only one chunk
